# Panoptica i18n Snapshot Log

This log tracks every rollback point taken during the internationalization (i18n)
extraction project. Each row points to a complete project snapshot you can
restore to if something goes wrong.

## How to read this

- **Tag** is what you'd say to Claude to roll back: e.g., "restore to 00-before-extraction".
- **NAS path** is on the QNAP at `//192.168.11.11/panoptica_backups/`.
- **Phase** describes what was about to happen next (or what just finished).

## How to roll back

1. SSH to the Panoptica VM.
2. `cd /opt/panoptica` (or wherever your live tree is).
3. Stop pm2: `pm2 stop panoptica`.
4. Rsync from the snapshot back, e.g.:
   `sudo rsync -a --delete --exclude='node_modules/' /mnt/qnap/2026-04-29_11-30-09/ /opt/panoptica/`
   (Ask Claude to give you the exact command for the snapshot you pick — it'll set up the right `--exclude` list.)
5. `pm2 start panoptica` and verify.

The `--exclude='node_modules/'` matters because the backup script doesn't capture node_modules; you keep your existing one.
